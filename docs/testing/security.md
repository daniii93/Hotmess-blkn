# HOTMESS Security Test Checklist

- [ ] RLS aktiv auf allen Tabellen.
- [ ] Admin Routes serverseitig geschuetzt.
- [ ] Scanner Routes serverseitig geschuetzt.
- [ ] Service Role Key niemals im Client Bundle.
- [ ] QR Signatur nicht manipulierbar.
- [ ] Ticket nicht doppelt scanbar.
- [ ] User sieht nur eigene Orders.
- [ ] User sieht nur erlaubte Chats.
- [ ] Verification Bucket privat.
- [ ] Webhooks pruefen Signatur.
- [ ] Webhooks sind idempotent.
- [ ] Webhooks schreiben Audit Log.
- [ ] Keine doppelten Tickets.
- [ ] Keine doppelten E-Mails.
